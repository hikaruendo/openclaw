# MAIN_MANAGER_GUIDE.md

## 基本運用

- 普段は `main` に話す
- 分解・委譲が必要な仕事は `manager` に振る
- `manager` は必要に応じて `radar / forge / scribe / ops` を使う

## そのまま使える依頼文

### 事業アイデア評価
- managerに、この事業アイデアを評価して。必要ならradarとforgeを使って、最後に今日の1手まで出して。

### 市場調査つきの判断
- managerに、この市場に入る価値があるか見て。radarで競合と需要を調べて、forgeで勝ち筋を整理して。

### X投稿作成
- managerに、この進捗をX投稿にして。必要ならscribeを使って、本命1案に絞って。

### 今日やること整理
- managerに、今やるべきことを整理して。必要ならopsを使って、90分で終わるタスクに分解して。

## コツ

- 目的を書く
- 必要なら使うagentを書く
- 最後に何が欲しいかを書く
  - 今日の1手
  - 本命1案
  - Go / No-Go判定
  - 90分タスク

## ルール

- 迷ったらまず `manager`
- 調査だけなら `radar` 直打ちでもOK
- 文章だけなら `scribe` 直打ちでもOK
- ただし複数論点が絡むなら `manager` 経由の方が安定
